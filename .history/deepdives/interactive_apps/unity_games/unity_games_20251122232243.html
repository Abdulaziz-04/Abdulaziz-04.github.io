Unity Gameplay Suite

A compact suite of four Unity projects that highlight core gameplay programming, game loops, and clean scene management. Focus on readable C#, reusable components, and clear player feedback. Each project ships with a short gameplay demo.

  • Tech Stack: Unity, C#, Animator, NavMesh, ScriptableObjects, UI Toolkit

Quick insights

  • Goal: Demonstrate gameplay engineering across tower defense, First Person Shooter and arcade physics.

  • What stands out: Clear scene flows, Object-oriented programming, modular scripts, and UI feedback loops.

  • What you will see: Short videos, concise architecture notes, and the key systems that drive each game.

Glitch Garden (Plants vs Zombies style) — Hero

  • Core loop: Place defenders on a grid with sun as currency while different attackers come down the lanes.

  • Key systems: LevelController for win and loss states, GameTimer to end spawns, lane-based Spawner, Resource store, DefenderSelector and DefenderSpawner UI.

  • Combat: Defenders detect attackers in lane and fire Projectiles. Attackers advance, animate, and damage via Health handlers.

  • UX polish: Scene Manager for splash and menu flow, Music Manager singleton object for persistent audio, GameHealth Manager for hearts and win or lose overlays.

  • Unity features used: Animator-driven attacks and deaths, prefab spawners, Scriptable separation of data where useful, audio mixing, and simple VFX.

FPS Survivor — three weapons, AI waves

  • Note: Built by following a FreeCodeCamp tutorial to explore FPS architecture and AI patterns in Unity.

  • Player systems: CharacterController movement, LookMovement for mouse aim, SprintCrouch with stamina, PlayerAttack handling hitscan rays and projectiles.

  • Weapons: WeaponManager switches WeaponHandler components. Handlers drive fire modes, aim states, muzzle VFX, and SFX.

  • Enemies: EnemyController finite state machine with Patrol, Chase, Attack using NavMeshAgent.EnemyAnimations and EnemySounds wrapper scripts.

  • Spawning and lifecycle: Spawner caps counts and respawns on enemy death. HealthMgmt applies damage, triggers animations and SFX, and cleans up objects.

Space Defender — wave shooter with pathing

  • Core loop: Player dodges and fires while enemies spawn in waves along predefined waypoint paths.

  • Wave design: WaveConfig ScriptableObjects define enemy prefab, count, spawn rate, and movement. WaveSpawner selects configs and instantiates at runtime.

  • Combat: Player fires lasers with Damage components. Enemy fires on timers and awards score on death. LaserShredder cleans off-screen projectiles.

  • UX and flow: SceneMgmt for menu and end states, MusicMgmt and GameSession as singletons, ScoreMgmt and HealthMgmt update UI in real time. BackGroundScroll animates space movement.

Block Breaker — classic arcade with scene progression

  • Core loop: Anchor ball to paddle until first click, then break blocks and advance levels. Missed ball triggers game over scene.

  • Key scripts: Initiate anchors and launches the ball with light randomness. Paddlemovement clamps input within camera bounds. Blockdestroy tracks hits, swaps sprites, spawns sparkles, and updates score and counters.

  • Progression: Level counts breakables, persists score, and triggers Sceneloader to load next scenes. ScoreDisplay shows totals on game-over. Speed slider adjusts global time scale for slow or fast play.

  • Unity features used: Collision events, AudioSource SFX, scene management, DontDestroyOnLoad for persistent state.

Why it matters

  • Breadth: Shows fluency across multiple gameplay patterns including tower defense, FPS, wave shooter, and arcade physics.

  • Engineering habits: Consistent scene flow, state management, and modular components that scale to new mechanics.

  • Player experience: Focus on readable feedback through UI, audio cues, animations, and reliable inputs.
